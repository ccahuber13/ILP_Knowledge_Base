Rebooting remote labs

Use soft and hard reboot in situations where the remote lab needs to reboot, such as newly installed software that requires a system restart.

To reboot a remote lab, follow the steps below:

  1. Click the Lab tab from the navigation bar.
  2. Click Actions, then choose a reboot option.
    • Select Soft reboot… to gracefully restart the remote lab. Soft reboot waits for the operating system to close all running applications.
    • Select Hard reboot… to power the remote lab off then on without waiting for running applications to close.
  3. Select the check-box next to each student whom you want to reboot their remote labs.
  4. Click OK.
Always attempt a soft reboot first as hard reboot has the potential to cause data loss. Only use hard reboot in situations where the remote lab does not respond to a soft reboot.

A cloud-based lab takes about a minute or less to reboot. A hardware-based lab can take 5 – 10 minutes to reboot depending on the complexity of the lab environment. The status panel on the Lab page displays a green upward facing arrow icon indicating you can connect to the remote lab.

If the status panel continues to display a red downward facing arrow after 10 minutes, see viewing system logs for more information on checking remote lab errors.
